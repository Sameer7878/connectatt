How does our website work?

What is a website, anyhow?

A web page is a way to display information on the internet. It’s made up of elements like text, images, links, videos, or buttons.

Based on the information those pages contain, they are organized into an information hierarchy– this allows navigation from one page to another. The overall collection of those related web pages is a website.

A website isn’t an application. It’s not a search engine. (Though a website might contain those things.) A website, at its heart, is just a way to publicly collect and display information. No matter how complicated a website gets, it all boils down to that basic purpose.

What is a website made of? How does a web browser work?

  • HTML(Hypertext Markup Language)code is a programming language that allows a web developer to design a web page. All of those pages contain elements like text, images, links, videos, or buttons., which are all written in code.
  • When you access a website on your computer using a browser. There are several different kinds of web browsers… Safari, Firefox, and Chrome are probably the most popular.
  • Regardless of which browser you use, It takes the code that the web developer has written out and decodes it into what you see when you type in a web address.
  • If your browser is too old to understand the code, it doesn’t translate the website properly. This is why new websites can look different or fail to work entirely on old computers.

Where is a website stored? What is web hosting?

  • This is where the concept of server comes as all the data related to a website is being stored on the servers which may contain all the required files in it and predefined code.
  • We have been using Selenium which is an open source tool that is used for automating the test cases carried out on web browsers or the web applications that are being tested using any web browser.
  • Using automation we extract the required information from the college website and this information is stored in an attribute at the back-end of our website.
  • We have been using flask as the web framework which renders the relevant information to a website and acts like the back-end for our web application.
  • When coming to web hosting we have chosen Heroku as the cloud where we have deployed our website because It is a free cloud provider maintained by salesforce.
